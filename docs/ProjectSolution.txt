# TechBlitz26 - Intelligent Sales Agent System - Complete Context

## Project Overview
Build an AI-powered multi-agent system that captures leads from multiple sources, automatically researches and scores them, notifies sales reps via WhatsApp/Telegram for approval, and handles complete outreach and follow-up autonomously across messaging, email, and voice calls.

## Core Problem Statement
Small and mid-size businesses lose leads daily because:
- Slow response times (hours/days instead of minutes)
- Inconsistent follow-up (reps forget after first attempt)
- Manual CRM maintenance (goes unmaintained)
- No pipeline visibility without physically checking
- Leads arrive via website, Instagram, WhatsApp while rep is busy

Result: Silent revenue leakage every single day.

## Mandatory Requirements

### 1. Multi-Source Lead Capture
- Website forms (embedded widget)
- Instagram DMs (Meta Graph API)
- WhatsApp Business messages (Meta Cloud API)
- Email inquiries (forwarding or IMAP)
- Voice calls (Twilio)
- Manual lead entry (dashboard)

### 2. Automatic Lead Research & Scoring
- Research agent searches web for legitimacy
- Checks company/person credibility
- Extracts budget indicators from conversation
- Analyzes urgency level and intent quality
- Assigns 0-10 score with reasoning
- Completes in <5 seconds

### 3. Rep Notification System
- WhatsApp or Telegram bot sends alert
- Simple approve/reject decision (buttons)
- Mobile-first (no laptop needed)
- Lead summary card with key details
- Real-time updates

### 4. Automated Outreach (Post-Approval)
- First response within 60 seconds
- Email + messaging multi-channel approach
- Personalized message generation (not robotic)
- Context-aware (references lead's specific inquiry)
- No manual prompting required

### 5. Intelligent Follow-Up System
- Automatic follow-ups if lead goes silent
- Timing optimization (learns best response times)
- Multi-sequence approach (Day 1, 2, 4, 7)
- Sentiment-aware messaging
- Stops if lead responds or asks to stop

### 6. Voice Call Integration
- AI can receive inbound calls (Twilio)
- Natural conversation using LLM
- Speech-to-text (Deepgram) + Text-to-speech (ElevenLabs)
- Transfers to human if complex
- Records and transcribes all calls

### 7. Pipeline Management
- Auto-logs all activities to database
- Rep notified only at key decision points
- Full control from phone via WhatsApp commands
- Real-time pipeline visibility
- Stage tracking (new → contacted → qualified → proposal → won/lost)

## Architecture Requirements

### Multi-Agent System (LangGraph)
Must implement 9 specialized agents with clear routing:

1. **Intake Agent**
   - Purpose: Normalize incoming leads from all channels
   - Tools: WhatsApp parser, Instagram parser, Email parser, Form parser
   - Output: Structured lead object
   - Routes to: Research Agent

2. **Research Agent**
   - Purpose: Gather intelligence on lead
   - Tools: Web search (Serper), Company lookup, Social scraper, Email validator
   - Reasoning: Chain-of-thought legitimacy check
   - Output: Research results with confidence score
   - Routes to: Scoring Agent

3. **Scoring Agent**
   - Purpose: Evaluate lead quality (0-10 score)
   - Tools: ML model, Rule-based system, Budget estimator
   - Reasoning: Multi-criteria decision analysis
   - Output: Score + detailed reasoning
   - Routes to: Notification Agent

4. **Notification Agent**
   - Purpose: Alert rep with context
   - Tools: WhatsApp Business API, Telegram Bot API, SMS backup
   - Waits for: Rep approval/rejection via button click
   - Routes to: Outreach Agent (if approved) or Archive (if rejected)

5. **Outreach Agent**
   - Purpose: Initial contact with approved lead
   - Tools: Email service (SendGrid), WhatsApp messaging, Instagram DM
   - Reasoning: Personalization engine (analyzes tone, context, urgency)
   - Output: Personalized first message
   - Routes to: Conversation Agent

6. **Conversation Agent**
   - Purpose: Handle ongoing dialogue with lead
   - Tools: LLM for natural language, FAQ knowledge base, Product catalog, Availability checker
   - Reasoning: Intent classification (question/objection/ready-to-buy/complex)
   - Memory: Full conversation history via vector DB
   - Routes to: Follow-up Agent / Voice Agent / Pipeline Agent / Rep escalation

7. **Follow-Up Agent**
   - Purpose: Re-engage silent leads
   - Tools: Scheduling engine, Message sequencer, Engagement tracker
   - Reasoning: Timing optimization (analyzes response patterns, timezone)
   - Routes to: Conversation Agent (when lead responds)

8. **Voice Agent**
   - Purpose: Handle phone calls
   - Tools: Twilio Voice, Deepgram STT, ElevenLabs TTS
   - Reasoning: Conversation flow manager, sentiment detector
   - Routes to: Pipeline Agent (after call) or Rep transfer

9. **Pipeline Agent**
   - Purpose: Update CRM and notify rep of progress
   - Tools: Database updater, CRM API, Analytics engine
   - Reasoning: Stage classification, close probability estimation
   - Routes to: Follow-up Agent or Close (won/lost)

### Agent Orchestration (LangGraph)
```python
State Machine Flow:
Intake → Research → Scoring → Notification → [Wait for approval]
  ↓ (if approved)
Outreach → Conversation ↔ Follow-up
  ↓
Voice (optional) → Pipeline → Close

Conditional Routing:
- If score < 5: Auto-reject, notify rep
- If score 5-7: Notify rep, wait for decision
- If score > 7: Notify rep as high-priority
- If conversation intent = "ready_to_buy": Route to Pipeline
- If conversation intent = "needs_call": Route to Voice
- If no response 48hrs: Route to Follow-up
```

### Agent Reasoning Requirements
Each agent must use Chain-of-Thought reasoning:
```
Example (Research Agent):
Thought: I need to verify if this is a legitimate business
Action: Search company name on Google
Observation: Found LinkedIn with 500+ employees
Thought: Appears legitimate, check domain age
Action: Query domain registry
Observation: Domain registered 5 years ago
Thought: Established company, likely has budget
Conclusion: High legitimacy score (8/10)
```

## WhatsApp Integration Details

### Setup Requirements
1. Facebook Business Manager account
2. WhatsApp Business API access
3. Verified business
4. Webhook endpoint (HTTPS required)

### Webhook Flow
```
Customer Message → Meta WhatsApp Server
  ↓
POST https://yourapi.com/webhooks/whatsapp
  ↓
Intake Agent processes
  ↓
Response back to Meta API
  ↓
Customer receives reply
```
